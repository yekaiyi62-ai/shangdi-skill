# 团队Skill编排模式

> 多个Skill协同工作的设计模式参考。

## 一、团队架构模式

### 模式1：星型（总调度器）

```
          用户
           ↕
        协调器Skill
       ↙   ↓   ↘
    成员A  成员B  成员C
```

**适用场景**：成员之间独立工作，由协调器统一分发和汇总。
**示例**：雅思备考团队（用户找协调器，协调器分发给口语/写作/听力教练）。

协调器职责：
- 接收用户输入，判断应该转给哪个成员
- 维护用户档案和进度
- 汇总各成员的反馈，生成综合报告
- 协调学习计划和优先级

### 模式2：流水线

```
用户 → 成员A → 成员B → 成员C → 输出
```

**适用场景**：每个步骤的输出是下一步的输入。
**示例**：内容生产线（选题→写作→审核→排版→发布）。

### 模式3：并行独立

```
用户 → 成员A → 输出A
用户 → 成员B → 输出B
用户 → 成员C → 输出C
```

**适用场景**：每个成员完全独立工作，用户分别调用。
**示例**：公司职能团队（每个部门独立运作）。

### 模式4：混合模式

```
        用户
         ↕
      协调器
      ↙    ↘
   成员A   成员B → 成员C（B的输出给C）
```

**适用场景**：部分成员独立，部分有依赖关系。

## 二、共享数据设计

### 用户档案 (user-profile.md)

团队成员共享的用户信息：

```markdown
# 用户档案

## 基本信息
- 目标：[用户的总目标]
- 当前水平：[评估结果]
- 时间规划：[截止日期、每天可用时间]

## 各维度评估
| 维度 | 当前水平 | 目标水平 | 最近评估日期 |
|------|---------|---------|------------|
| [维度1] | [分数] | [目标] | [日期] |
| [维度2] | [分数] | [目标] | [日期] |

## 学习偏好
- 偏好的学习方式：[视觉/听觉/实操]
- 薄弱环节：[列表]
- 已掌握：[列表]
```

### 进度追踪 (progress.md)

```markdown
# 学习进度

## 总进度
- 开始日期：[日期]
- 当前阶段：[阶段名]
- 总完成率：[百分比]

## 各模块进度
| 模块 | 已完成任务 | 总任务 | 最近活动 |
|------|----------|-------|---------|
| [模块1] | N | M | [日期：做了什么] |

## 练习记录
| 日期 | 模块 | 内容 | 得分 | 备注 |
|------|------|------|------|------|
| [日期] | [模块] | [练习内容] | [分数] | [改进建议] |
```

### 共享数据读写规则

1. **协调器有写权限**：负责创建和更新共享数据
2. **成员有读权限 + 追加权限**：可以读取共享数据，可以追加练习记录
3. **冲突处理**：以最新时间戳为准
4. **数据位置**：统一存储在 `shared/` 目录下

## 三、协调器SKILL.md的核心结构

```markdown
## 问题路由

收到用户输入后，先判断应该转给哪个成员：

| 用户意图 | 转发给 | 触发词 |
|---------|--------|-------|
| [意图1] | 成员A | [关键词列表] |
| [意图2] | 成员B | [关键词列表] |
| [综合查询] | 协调器自己处理 | [关键词列表] |

## 转发规则

转发给成员时，必须附带：
1. 用户的原始输入
2. 用户档案摘要（从shared/user-profile.md读取）
3. 相关的历史记录（从shared/progress.md读取）

## 汇总报告

用户要求查看总进度时：
1. 读取 shared/progress.md
2. 读取各成员最近的练习记录
3. 生成综合报告，包含：各维度进度、薄弱环节、下阶段建议
```

## 四、成员Skill的团队适配

每个成员Skill除了自身的功能逻辑，还需要：

1. **读取共享数据**：启动时自动读取 `shared/user-profile.md`，了解用户情况
2. **记录活动**：完成一次练习/评估后，追加记录到 `shared/progress.md`
3. **标准化输出**：输出格式统一，方便协调器汇总

```markdown
## 团队协作规则（在成员SKILL.md中）

### 启动时
1. 读取 `shared/user-profile.md`，了解用户当前水平和目标
2. 读取 `shared/progress.md`，了解最近练习情况

### 完成后
1. 将本次练习/评估结果追加到 `shared/progress.md`
2. 如果发现用户水平有显著变化，更新 `shared/user-profile.md`
```

## 五、团队规模建议

| 团队规模 | 适用场景 | 注意事项 |
|---------|---------|---------|
| 2-3个Skill | 简单协作 | 可以不需要专门的协调器 |
| 4-6个Skill | 标准团队 | 需要协调器，推荐星型模式 |
| 7-10个Skill | 大型团队 | 考虑分组，每组一个子协调器 |
| >10个 | 不推荐 | 复杂度爆炸，拆分为多个独立团队 |
