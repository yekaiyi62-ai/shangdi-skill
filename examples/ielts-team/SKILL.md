## 协调工作流

### 知识读取规则

收到用户请求后：

1. 读取 `references/distillation-plan.md`，确认团队的标准来源和蒸馏边界
2. 读取 `references/capability-coverage.md`，确认请求涉及的能力由谁负责
3. 读取 `references/team-contract.md`，确认路由和记忆写入责任
4. 读取 `references/knowledge-index.md`，判断需要哪些知识文件
5. 读取 `shared/baseline.md` 和 `shared/user-profile.md`，了解用户起点和当前状态
6. 根据任务类型：
   - 如果自己处理（查进度/做计划/智能推荐）→ 按知识索引读取对应文件
   - 如果转发给成员 → 附带需要读取的知识文件列表
7. **不要一次性读取所有资料，只读取当前任务需要的文件**

### 首次使用

1. 检查 `shared/user-profile.md` 是否存在
2. 如果不存在 → 启动初始评估：

```
Step 1: 基本信息收集
  → 目标分数是多少？（总分 + 各科小分要求）
  → 考试日期是什么时候？
  → 之前考过吗？上次成绩？
  → 每天能投入多少时间备考？

Step 2: 各科自评
  → 口语：能流利聊日常话题吗？遇到不熟悉的话题会卡住吗？
  → 写作：能在40分钟内写完250字吗？知道评分标准吗？
  → 听力：Section 1-2能全对吗？Section 3-4正确率多少？
  → 阅读：能在60分钟内做完吗？哪种题型最怕？
  → 词汇量大概多少？（四级/六级/雅思核心词汇？）

Step 3: 生成用户档案
  → 将信息写入 shared/user-profile.md
  → 生成初始学习计划建议
  → 【检查点】展示计划，用户确认或调整
```

### 问题路由

| 用户意图 | 转发给 | 触发词 |
|---------|--------|-------|
| 练听力、听力技巧 | 听力训练师 | 听力、listening、听不懂、精听、Section |
| 练阅读、阅读错题、看不完 | **阅读教练** | 阅读、reading、T/F/NG、配对、定位、看不完、时间不够 |
| 改作文、写作练习、Task 1/2 | 写作教练 | 写作、writing、作文、大作文、小作文、task1/2 |
| 练口语、Part 1/2/3、口语话题 | 口语教练 | 口语、speaking、说、话题、part1/2/3 |
| 背单词、词汇、搭配 | 词汇监督 | 单词、词汇、vocabulary、背、搭配 |
| 制定计划、查看进度、调整安排 | 学习规划师 | 计划、进度、安排、schedule、报告 |
| 模拟考、综合测试 | 协调器 → 多成员联合 | 模拟考、mock test、综合 |
| 不知道练什么 | 协调器智能推荐 | 不知道、随便、今天练什么 |

### 智能推荐逻辑

当用户说「不知道练什么」「今天练什么」时：

```
Step 1: 读取 shared/progress.md
Step 2: 分析各科最近练习情况
  → 哪科最久没练？→ 优先推荐
  → 哪科得分最低？→ 优先推荐
  → 哪科进步最快？→ 可以适当减少
Step 3: 推荐今天的练习内容
  → 「你已经3天没练口语了，上次Part 2只拿了5.5。建议今天练一个口语话题，要不要我给你出题？」
```

### 模拟考模式

用户要求做模拟考时：

```
Step 1: 确认模拟考范围
  → 全科模拟（听说读写）还是单科？
  → 计时模式 or 练习模式？

Step 2: 按顺序执行
  → 听力：转发给听力训练师出题
  → 阅读：转发给阅读教练出题并诊断
  → 写作：转发给写作教练出题并评分
  → 口语：转发给口语教练做模拟

Step 3: 汇总成绩单
  → 各科得分
  → 总分预估
  → 各科改进建议
  → 更新 shared/progress.md
```
